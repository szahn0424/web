리스트

홈페이지에서 구조와 형태가 비슷한 연관되어있는 컨텐츠를 표기하는 방법

ordered list - ol

연관된 컨텐츠 리스트로 표현되는 컨텐츠사이에 순서의 영향이 있는 경우 ol태그를 부모태그로 사용한다

  1. git add
  2. git commit-m "메세지"
  3. 깃 푸쉬 오리진 마스털
  1. 물끓이기
  2. 스프넣는당
  3. 면도넣는당

unordered list -ul

연관되어있는 컨텐츠 즉 li태크들이 서로 순서의 영향이 없을때 부모태그로 ul태그를 사용한다

li태그는 단독사용 안됨, 반드시 순서흐름 고려해 순서있으면 부모태그에 ol태그를 없다면 부모태그에 ul태그를 사용해야함

ol/ul태그의 자식태그로는 li태그만 사용할 수 있다(h1~~태그 같은거 쓸 수 없당)

li태그안에는 또 다른 문맥들이 형성될수있덩..제목 설명태그 등등 여러가지 태그문맥이 li 태그 안에서 사용된당
